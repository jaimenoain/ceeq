## Supabase/Resend Auth Loop – Complete

- **Supabase client utilities:** `lib/supabase/client.ts` (browser, `createBrowserClient`) and `lib/supabase/server.ts` (server, `createServerClient` with Next.js `cookies()`) use `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`. Middleware (`middleware.ts`) refreshes the auth session on each request and writes cookies to the response.
- **Server Actions:** `app/actions/auth.ts` implements login, signup, forgot password (reset email), and logout. All use the server Supabase client and redirect or return errors. Input validation matches DOMAIN_MODEL: `email` (valid email), `password` (min 8 characters, message: "Password must be at least 8 characters").
- **Login page:** `/login` uses Shadcn Card, Tabs (Log in / Sign up / Forgot password), Input, Button, and inline labels. Forms are wired to the auth Server Actions; errors and success (forgot password) display inline. The `(auth)` layout remains minimal.
- **Protected routes:** Unauthenticated users hitting `/` are redirected to `/login`. Dashboard routes under `(dashboard)` are protected in `app/(dashboard)/layout.tsx`: `supabase.auth.getUser()` is awaited; if no user, redirect to `/login`. Authenticated users can access the dashboard; the sidebar Logout button submits a form that calls `logoutAction`, which signs out and redirects to `/login`.
- **Tenant isolation:** RLS and tenant-scoped access rely on `auth.jwt().tenant_id` in the database. Ensure the Supabase project sets a custom claim `tenant_id` (e.g. via Auth hooks or `app_metadata`) when users sign up or are assigned to a tenant so RLS policies can enforce isolation.
- **Resend (email provider):** To use Resend for auth emails (signup confirmation, password reset), configure Supabase to use a custom SMTP provider: In the **Supabase Dashboard** go to **Project → Authentication → Providers → Email**, then under **SMTP Settings** (or **Custom SMTP**) set the SMTP server to Resend (e.g. `smtp.resend.com`, port 465/587) and use your Resend API key as the password. Enable the email provider and, if desired, disable "Confirm email" for development.
