## App Router – Route Groups

- **Route groups** separate unauthenticated and authenticated experiences; group names `(auth)` and `(dashboard)` do not affect the URL path.
- **File tree (new routing structure):**

```
app/
├── (auth)/
│   └── login/
│       └── page.tsx          → /login
├── (dashboard)/
│   ├── repository/
│   │   └── page.tsx          → /repository
│   ├── pipeline/
│   │   ├── page.tsx          → /pipeline
│   │   └── closed/
│   │       └── page.tsx      → /pipeline/closed
│   ├── deals/
│   │   └── [id]/
│   │       └── page.tsx      → /deals/[id]
│   ├── tasks/
│   │   └── page.tsx          → /tasks
│   └── settings/
│       └── page.tsx          → /settings
├── globals.css
├── layout.tsx
├── page.tsx                  → /
└── favicon.ico
```

- **Paths:** `/login` (auth), `/repository`, `/pipeline`, `/pipeline/closed`, `/deals/[id]`, `/tasks`, `/settings` (dashboard). All new pages are placeholders with a single semantic `<h1>` only. Layout-level auth checks can be added later to `(dashboard)/layout.tsx` and `(auth)/layout.tsx` as needed.
